# Proprietary blobs for Ayaneo Pocket DS (SM8750 / TurboX C8550)
# Format: [- ]<path_on_device>[|sha1sum]
#   Lines starting with '-' are pinned (not updated during extract)
#   |sha1sum suffix pins to a specific file version
#
# Derived from LineageOS android_device_oneplus_sm8750-common (lineage-23.0)
# with OnePlus/OPLUS-specific blobs removed. Platform codename: sun
#
# Reference: LineageOS/android_device_oneplus_sm8750-common
#
# ============================================================
# HOW TO VERIFY AND UPDATE BLOBS AGAINST STOCK FIRMWARE
# ============================================================
#
# All blob paths in this file are UNVERIFIED placeholders derived from the
# upstream SM8750 ("sun") reference device (OnePlus 13 "dodge"). They must
# be validated against the actual Ayaneo stock partitions before building.
#
# Step 1 — Mount stock partitions
#   Use simg2img + mount, or unpack super with lpunpack:
#     simg2img super_1.img..super_8.img super_raw.img
#     lpunpack super_raw.img out/
#     mount -o ro out/vendor.img /mnt/vendor
#     mount -o ro out/system.img /mnt/system
#     mount -o ro out/odm.img /mnt/odm        # if present
#
# Step 2 — Run catalog-blobs.sh
#     scripts/catalog-blobs.sh \
#         --vendor /mnt/vendor \
#         --system /mnt/system \
#         --odm    /mnt/odm \
#         --out    device/ayaneo/pocket_ds/proprietary-files.txt
#
#   The script emits one path per line, alphabetically within each
#   partition prefix (vendor/, system/, system_ext/, odm/), and can
#   optionally append sha1sum pins with --pin-sha1.
#
# Step 3 — Run extract_utils
#   After verifying paths, extract blobs into vendor/ayaneo/pocket_ds/:
#     ./extract-files.sh /mnt
#
# Blobs marked with '-' are pinned and will NOT be updated by extract-files.
# Blobs with a '|sha1' suffix are version-locked to that exact binary.
# ============================================================
